    Logic Explanation

    Section 1: Original Features

    My application stores all loot items inside an array called lootItems. Each time the user adds loot, I create an object with a name and value and push it into the array. I use a traditional for loop inside renderLoot() to calculate the running total and to build the HTML list of items. I also use another loop inside splitLoot() to calculate the final total before dividing it by the party size. Conditional logic is used throughout the app to validate inputs, such as checking for empty loot names, invalid numbers, negative values, missing party size, or trying to split loot when no items exist. All validation messages appear inside the interface instead of using alerts.

    Section 2: Bonus Features

    I implemented two optional bonus features. First, the application automatically recalculates the loot split whenever a new loot item is added, as long as the party size is valid. This makes the interface more dynamic and responsive. Second, I added a guild tax system: if the total loot exceeds 100 gold, a 10% tax is applied before splitting the loot. The tax amount is displayed to the user. These features enhance the realism and interactivity of the application.
